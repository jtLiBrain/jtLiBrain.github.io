Scala项目代码质量

Posted on 2020-11-27 Edited on 2020-12-06 In code quality , scala Views:

本文基于Scala和Java进行混合编程的Maven项目介绍搭建代码质量设施，本文旨在快速理解以及搭建过程，并不侧重介绍每个工具的具体使用方法，详细参见各相关文档。

构建代码风格设施

Java风格

Maven Checkstyle

Checkstyle插件根据开发人员所使用的代码风格生成报告。其内预定义了两个规则集：sun_checks.xml和 google_checks.xml。

参见介绍。

运行Checkstyle

使用Checkstyle Maven插件有三种方式，参见用法：

  • Checkstyle报告作为项目所有报告的一部分，借助<reporting/>元素和mvn site命令；
  • 如果不需要进行特别的配置，可以直接运行mvn checkstyle:checkstyle命令来生成Checkstyle报告，而不需要在pom.xml中指定Checkstyle插件；
  • 将Checkstyle绑定到项目build过程，这样可以使用Checkstyle的检查结果来决定项目构建是否进行；

style配置文件，Scalastyle为我们提供了一个默认的配置scalastyle_config.xml，将其保存至项目的style/路径下。

  • 关于如何自定义规则，参考自定义规则；

Java测试覆盖率

JaCoCo(Java Code Coverage)

JaCoCo是一个免费的Java代码覆盖率类库，它执行Eclipse Public License。

Maven插件

JaCoCo Maven插件向测试用例提供了JaCoco运行时，可以创建测试报告。该插件提供了很多Maven goal，使用mvn help:describe -Dplugin=org.jacoco:jacoco-maven-plugin -Ddetail可以列出它所提供的goal。

运行mvn clean test jacoco:report：

SCoverage是一个Scala代码覆盖率工具，它执行Apache License，提供语句和分支的覆盖率。

Maven插件

运行：mvn clean test scoverage:report：
